# PetCircle Authentication Pages - UI Design Specification

## Document Overview
This document provides comprehensive UI/UX specifications for the authentication flow of PetCircle, a community platform for pet owners. It includes design requirements for AI design tools and implementation guidelines for frontend developers.

---

## 1. Layout Architecture

### 1.1 Split-Screen Design
- **Layout Type**: Two-column split screen (50/50 on desktop)
- **Left Panel**: Hero carousel with feature slides
- **Right Panel**: Authentication forms
- **Responsive Behavior**:
  - Desktop (>1024px): 50/50 split
  - Tablet (768px-1024px): 40/60 split (carousel/form)
  - Mobile (<768px): Stacked layout, carousel above form

### 1.2 Container Structure
```
┌─────────────────────────────────────────────┐
│  Left Panel (Carousel)  │  Right Panel      │
│                          │  (Forms)          │
│  [Feature Slides]        │  [Auth Forms]     │
│                          │                   │
│  • Slide Navigation      │  • Dynamic Form   │
│  • Auto-rotation         │  • Form Switching │
└─────────────────────────────────────────────┘
```

## 2. Left Panel - Feature Carousel

### 2.1 Carousel Specifications

**Dimensions:**
- Width: 50% of viewport (desktop)
- Height: 100vh (full viewport height)
- Fixed position during scroll

**Behavior:**
- Auto-rotation: 5 seconds per slide
- Manual navigation: Dot indicators at bottom
- Smooth transitions: 800ms ease-in-out
- Pause on hover
- Swipe gestures enabled (touch devices)

**Overlay:**
- Dark gradient overlay: linear-gradient(rgba(0,0,0,0.3), rgba(0,0,0,0.5))

## 12. Validation Rules

### 12.1 Client-Side Validation

**Email:**
- Format: RFC 5322 compliant
- Real-time validation on blur
- Error: "Please enter a valid email address"

**Password:**
- Min length: 8 characters
- Max length: 128 characters
- Must contain:
  - 1 uppercase letter
  - 1 lowercase letter
  - 1 number
  - 1 special character
- Real-time strength indicator
- Error messages:
  - "Password must be at least 8 characters"
  - "Include at least one uppercase letter"
  - "Include at least one number"
  - "Include at least one special character"

**Name Fields:**
- Min length: 2 characters
- Max length: 50 characters
- Only letters, spaces, hyphens
- Error: "Please enter a valid name"

**Verification Code:**
- Exactly 6 digits
- Auto-submit when complete
- Error: "Invalid verification code"

### 12.2 Form Submission

**Loading States:**
- Disable all inputs
- Show spinner in button
- Prevent multiple submissions

**Error Handling:**
- Display errors below relevant fields
- Scroll to first error
- Focus first error field
- Generic error: Toast notification at top

**Success Handling:**
- Success message or redirect
- Clear form if staying on page
- Auto-redirect delay: 2 seconds

## 13. Accessibility Requirements

### 13.1 ARIA Labels

```html
All inputs:
- aria-label or associated label
- aria-required="true"
- aria-invalid="true/false"
- aria-describedby for error messages

Buttons:
- aria-label for icon buttons
- aria-busy="true" when loading

Form:
- role="form"
- aria-labelledby for form heading
```

### 13.2 Keyboard Navigation

- Tab order: Logical top-to-bottom
- Enter: Submit form
- Escape: Close modals/overlays
- Arrow keys: Navigate carousel
- Space: Toggle checkboxes

### 13.3 Screen Reader Support

- Announce form errors
- Announce loading states
- Announce success messages
- Skip links for navigation
- Descriptive link text

### 13.4 Focus Management

- Visible focus indicators
- Focus trap in modals
- Return focus after actions
- Logical tab order

```javascript

{
  user: null,
  isAuthenticated: false,
  isLoading: false,
  error: null,
  currentForm: 'registration' // or 'login', 'verify', etc.
```

### 17.3 API Integration

```javascript

// Password reset request
POST /api/auth/password-reset/
Body: { email }

// Password reset confirm
POST /api/auth/password-reset-confirm/
```
