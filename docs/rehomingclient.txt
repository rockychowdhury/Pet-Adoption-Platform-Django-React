# Frontend Development Instructions: Rehoming Request Flow

## Overview
This document provides detailed UI/UX instructions for building the complete rehoming request flow. Each page should prioritize emotional support, clarity, and gentle guidance through a difficult decision.

---

## Page 1: Entry Point - "Rehome My Pet" Button

### Location
- Display in user dashboard or pet profile page
- Button should be subtle, not prominent (this shouldn't be encouraged)

### Visual Design
- Use a neutral color (not bright/action colors)
- Text: "Rehome My Pet" or "Find a New Home"
- Include a small info icon that shows tooltip: "We'll guide you through this process with care"

### On Click Behavior
- Trigger profile completeness check
- Show loading state while checking

---

## Page 2: Profile Completion Gate (Conditional)

### When to Show
Only display if user profile is incomplete

### Layout Structure

**Header Section:**
- Title: "Before We Begin"
- Subtitle: "We need a complete profile to ensure the best outcome for your pet"

**Content Section:**
- Show a checklist of missing profile fields with visual indicators:
  - ✓ Completed items (green checkmark)
  - ○ Missing items (empty circle or red icon)
- List missing fields clearly:
  - First Name
  - Last Name
  - Phone Number
  - Location (City, State)
  - Email Verification Status
  - [Any other required fields]

**Missing Fields Display:**
- Group related fields together
- Show friendly labels (not database field names)
- Use icons next to each requirement

**Call-to-Action:**
- Primary button: "Complete My Profile"
- Secondary link: "I'll do this later"

**Additional Elements:**
- Brief explanation: "A complete profile helps us verify you as a responsible owner and provides adopters with confidence"

---

## Page 3: Pet Selection

### Layout Structure

**Header:**
- Title: "Which pet do you need to rehome?"
- Subtitle: "Select the pet you're looking to find a new home for"

**Pet Grid/List:**
- Display all user's active pets (status = 'active', not already in rehoming)
- Each pet card should show:
  - Pet photo (prominent, circular or square with rounded corners)
  - Pet name (large, clear font)
  - Species and breed
  - Age
  - Profile completion indicator

**Pet Card Design:**
- Hover state: subtle elevation/shadow
- Click/tap should be obvious
- If pet profile incomplete, show yellow badge: "Profile Needs Completion"

**Empty State:**
- If no eligible pets: "You don't have any pets available for rehoming"
- Link to add a pet or return to dashboard

**Footer:**
- Cancel/Back button

## Page 4: Pet Profile Completion Gate (Conditional)

### When to Show
Only if selected pet's profile is incomplete

### Layout Structure

**Header:**
- Pet photo (large, centered)
- Pet name
- Title: "Let's Complete [Pet Name]'s Profile"
- Subtitle: "A complete profile helps find the perfect home"

**Profile Completion Checklist:**
- Visual checklist similar to user profile page
- Show completed vs. missing items:
  - ✓ Name
  - ✓ Species
  - ○ Breed (missing)
  - ○ Birth date or age estimate (missing)
  - ○ Gender (missing)
  - ○ Description (missing)
  - ○ Photo (missing)
  - [Other required fields]

**Missing Information Display:**
- Highlight what's needed in an easy-to-scan format
- Use friendly language, not technical terms

**Call-to-Action:**
- Primary button: "Complete [Pet Name]'s Profile"
- Secondary link: "Go Back"

**Informational Note:**
- "A detailed profile with photos increases adoption success by 3x" (or similar encouraging stat)

## Page 5: Emotional Check-In

### Purpose
Give users a moment to reflect before proceeding

### Layout Structure

**Visual Header (Full Width):**
- Large, high-quality photo of the pet (full bleed or with gentle vignette)
- Overlay with pet's name in elegant typography

**Main Content (Centered, Max-Width Container):**

**Emotional Message:**
- Use warm, empathetic tone
- Example text:
  ```
  "We Understand This Is Difficult"
  
  Rehoming a pet is one of the hardest decisions a pet owner can make. 
  We're here to support you through this process and help find the best 
  possible home for [Pet Name].
  
  Before we continue, we want to make sure you've considered all options:
  ```

**Consideration Prompts (Subtle Boxes or Cards):**
- "Have you explored pet-friendly housing options?"
- "Are there family or friends who could temporarily help?"
- "Have you contacted local pet support organizations?"
- Each with a small icon

**Resource Links:**
- "View rehoming alternatives" (expands to show resources)
- "Connect with pet counselors" (if available)

**Pet Summary Sidebar (or Card):**
- Pet photo (smaller)
- Name
- Age, breed, species
- Key personality traits

**Decision Buttons (Clear Visual Separation):**

**Option 1 - Primary (Styled as less prominent):**
- "I Need More Time to Decide"
- Subtext: "Save as draft and return later"
- Styling: Outline button or secondary color

**Option 2 - Secondary (Styled as more prominent):**
- "I'm Ready to Proceed"
- Subtext: "Continue with rehoming process"
- Styling: Solid button, forward-moving color

**Footer Note:**
- "Your draft will be saved and you can return anytime"

## Page 6: Rehoming Request Form

### Layout Structure
Multi-step form with progress indicator

**Progress Bar at Top:**
- Visual indicator showing: Step X of 5
- Steps labeled: Reason → Urgency → Location → Home Details → Privacy & Terms

---

### Step 1: Reason for Rehoming

**Header:**
- Step number and title: "Step 1 of 5: Why Are You Rehoming [Pet Name]?"

**Form Field:**
- Large textarea (minimum 4-5 rows visible)
- Placeholder: "Please share your situation. This helps us understand your needs and find the best match for your pet."
- Character counter showing: "0 / 50 minimum"
- Requirement note: "Minimum 50 characters required"

**Helper Text:**
- "Common reasons include: moving, allergies, financial hardship, behavioral challenges, lifestyle changes"
- "Be honest - this information helps match your pet with the right home"

**Validation:**
- Real-time character count
- Show error if under 50 characters on attempt to proceed
- Error message: "Please provide more detail (at least 50 characters)"

**Buttons:**
- "Back" (outline/secondary)
- "Continue" (primary)

---

### Step 2: Urgency Timeline

**Header:**
- "Step 2 of 5: What's Your Timeline?"

**Radio Button Options (Large, Card-Style):**

**Option 1: Immediate - Emergency**
- Icon: Urgent/Alert icon
- Title: "Immediate - Emergency Situation"
- Description: "I need to rehome within days (no waiting period)"
- Badge: "Your listing will go live immediately"

**Option 2: Within 1 Month**
- Icon: Calendar icon
- Title: "Within 1 Month"
- Description: "I can wait a few weeks to find the right home"
- Badge: "24-hour reflection period applies"

**Option 3: Flexible Timeline**
- Icon: Clock icon  
- Title: "Flexible Timeline"
- Description: "I have time to find the perfect match"
- Badge: "24-hour reflection period applies"

**Information Callout:**
- Light blue background box
- "Why a reflection period? It gives you 24 hours to reconsider and ensures you're certain about this decision. You can cancel anytime during this period."

**Buttons:**
- "Back"
- "Continue"

---

### Step 3: Location Details

**Header:**
- "Step 3 of 5: Where Is [Pet Name] Located?"

**Form Fields:**

**Auto-filled Section (If Available):**
- Show pre-filled data from user profile with light background
- Label: "From your profile:"
- City (pre-filled, editable)
- State (pre-filled, editable)
- ZIP Code (pre-filled, editable)

**Edit Override:**
- Checkbox: "Use different location for this pet"
- When checked, clear fields and allow manual entry

**Manual Entry Fields:**
- City (text input)
- State (dropdown)
- ZIP Code (text input, optional)

**Map Preview (Optional Enhancement):**
- Small embedded map showing general area (not exact address)
- Privacy note: "We only show your general area to adopters"

**Privacy Information Box:**
- "Your exact address is never shared. Adopters will only see your city and state."

**Buttons:**
- "Back"
- "Continue"

---

### Step 4: Ideal Home Description

**Header:**
- "Step 4 of 5: Describe the Ideal Home for [Pet Name]"
- Subtitle: "This helps us find the perfect match (optional but recommended)"

**Form Field:**
- Large textarea
- Placeholder: "Example: [Pet Name] would thrive in a home with a fenced yard, no small children, and an owner who works from home. They love other dogs but can be nervous around cats."

**Prompting Questions (Collapsible Helper):**
- "Consider including:" (expandable section)
  - Living situation (house, apartment, yard?)
  - Other pets compatibility
  - Children compatibility  
  - Activity level needed
  - Experience level required
  - Any special needs or requirements

**Character Guidance:**
- No minimum, but show encouraging text: "More detail helps find better matches"
- Optional character counter

**Skip Option:**
- Checkbox: "I'll add this later"

**Buttons:**
- "Back"
- "Continue"

---

### Step 5: Privacy & Terms

**Header:**
- "Step 5 of 5: Privacy and Agreement"

**Privacy Level Selection:**

**Section Title:** "Who Can See This Listing?"

**Radio Options (Card Style):**

**Option 1: Public**
- Icon: Globe or eye icon
- Title: "Public - Anyone Can See"
- Description: "Your listing will be visible to all visitors. This gives you the widest reach."

**Option 2: Verified Users Only**
- Icon: Shield or checkmark icon
- Title: "Verified Users Only"
- Description: "Only users who have completed verification can view your listing. More security, but fewer potential adopters."

**Information Note:**
- "Regardless of your choice, we never share your contact information publicly. Interested adopters contact you through our secure messaging system."

---

**Terms of Service Section:**

**Title:** "Rehoming Agreement"

**Terms Summary Box (Scrollable, Light Background):**
- Display key points in readable format:
  - Your responsibilities
  - Our platform's role
  - Privacy commitments
  - Liability disclaimers
  - [Other important terms]

**Checkbox (Required):**
- Large, clear checkbox
- Label: "I have read and agree to the Rehoming Terms of Service"
- Link: "Read Full Terms" (opens in new tab/modal)

**Visual Requirements:**
- Checkbox must be checked to enable "Submit" button
- Error state if trying to submit unchecked

---

**Review Summary (Collapsible Panel):**
- "Review Your Information" (expandable)
- When expanded, show summary of all previous steps:
  - Reason (first 100 characters)
  - Urgency level
  - Location
  - Ideal home notes (if provided)
  - Privacy level
- Each section with "Edit" link that jumps to that step

---

**Final Buttons:**
- "Back" (outline)
- "Submit Request" (primary, large, disabled until terms accepted)

---

## Page 7: Cooling Period Confirmation (Conditional)

### When to Show
After form submission, if urgency is NOT "immediate"

### Layout Structure

**Success Icon:**
- Large checkmark or hourglass icon (green/blue)
- Centered at top

**Header:**
- Title: "Your Request Has Been Received"
- Subtitle: "24-Hour Reflection Period Started"

**Main Content (Centered Card):**

**Explanation:**
```
"We've received your rehoming request for [Pet Name].

To ensure you're certain about this important decision, we've started 
a 24-hour reflection period. This gives you time to:

• Reconsider your decision
• Explore alternatives  
• Discuss with family
• Feel confident about moving forward

After 24 hours, you'll be able to confirm and publish your listing."
```

**Timeline Indicator:**
- Visual timeline showing:
  - "Now" (current time)
  - "24 Hours" (countdown or end time)
  - "Listing Goes Live" (if confirmed)

**Countdown Display:**
- Large, clear countdown timer
- "Time Remaining: 23 hours, 45 minutes"

**What Happens Next Section:**
- Numbered list or timeline:
  1. "Review your decision over the next 24 hours"
  2. "We'll email you when the period ends"
  3. "Confirm or cancel your request"
  4. "If confirmed, your listing goes live immediately"

**Action Buttons:**

**Primary Options:**
- "Return to Dashboard" (primary button)

**Secondary Option:**
- "Cancel This Request" (text link, red/warning color)
  - Clicking opens confirmation modal

**Email Notification Note:**
- Icon + text: "We've sent confirmation to [email]"
- "The email includes a link to cancel if you change your mind"

**Save Draft Option:**
- If user wants to edit: "Edit Request Details" link

### When to Show
User clicks "Confirm & Publish Listing" after cooling period

### Modal Layout

**Modal Size:** Medium, centered

**Header:**
- Title: "Confirm Publication"
- Close button (X)

**Content:**

**Final Check Message:**
```
"You're about to publish a public listing to rehome [Pet Name].

Are you sure you want to proceed?"
```

**Reminder Checklist:**
- ✓ "I've explored all alternatives"
- ✓ "I understand this listing will be public" (or "visible to verified users")
- ✓ "I'm ready to communicate with potential adopters"
- ✓ "I agree to the rehoming terms"

**Pet Summary:**
- Pet photo (small)
- Name, age, breed
- Location
- Privacy level

**Warning Box (Optional):**
- Soft yellow background
- "Once published, this will notify potential adopters. You can still pause or cancel your listing later."

**Modal Actions:**
- "Go Back" (secondary, left aligned)
- "Confirm & Publish" (primary, right aligned, green)

**After Confirmation:**
- Close modal
- Show success toast/notification
- Redirect to active listing page or dashboard
- Send confirmation email

## Universal UI Elements & Best Practices

### Navigation
- Always provide clear "Back" and "Cancel" options
- Show progress when in multi-step flows
- Breadcrumbs for complex flows

### Accessibility
- All buttons must have clear labels
- Form fields need visible labels and helpful error messages
- Color is not the only indicator (use icons + text)
- Keyboard navigation support
- Screen reader friendly text

### Responsive Design
- Mobile-first approach
- Forms should be single-column on mobile
- Large tap targets (minimum 44x44px)
- Readable font sizes (minimum 16px for body text)

### Loading States
- Show spinners/skeletons during data loading
- Disable buttons during processing
- Clear feedback when actions complete

### Error Handling
- Inline validation on form fields
- Clear error messages in plain language
- Error summary at top of form if multiple errors
- Suggest corrections when possible

### Emotional Design
- Use warm, empathetic language throughout
- Pet photos should be prominent and high-quality
- Avoid harsh colors or aggressive CTAs
- Celebrate small wins (profile completion, etc.)
- Provide encouragement and support

### Notifications
- Toast messages for quick feedback
- Email confirmations for important actions
- In-app notifications for status changes
- Clear notification preferences

### Color Psychology
- Success: Green (but not too bright)
- Warning/Reflection: Amber/Yellow
- Error/Cancel: Red (soft, not alarming)
- Primary actions: Blue or brand color
- Pet-related content: Warm, friendly colors
