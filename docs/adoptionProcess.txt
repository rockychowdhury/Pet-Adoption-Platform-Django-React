## ðŸ”„ DETAILED ADOPTION JOURNEY

### **STAGE 1: Application Submission**

#### **Applicant Actions:**
1. **Browse Available Pets**
   - View pet profiles with full details
   - Filter by species, age, breed, temperament
   - Read adoption requirements (listed per pet)

2. **Submit Application**
   - **Required Information:**
     - Personal details (name, email, phone, address)
     - Housing information:
       - Type: House/Apartment/Condo
       - Own or Rent (if rent, landlord approval required)
       - Yard size (if applicable)
       - Existing pets (species, age, vaccination status)
     - Household composition:
       - Number of adults/children
       - Ages of children
       - Allergies to pets
     - Pet experience:
       - Previous pet ownership
       - Experience with this species/breed
       - Training knowledge
     - Lifestyle assessment:
       - Work schedule (hours away from home)
       - Activity level (sedentary/moderate/active)
       - Exercise commitment
     - Veterinary reference (name, clinic, phone)
     - Personal references (2 required, non-family)
     - Why they want to adopt this specific pet (essay, 200+ words)
   
   - **Document Uploads (optional but recommended):**
     - Proof of residence
     - Landlord approval letter (if renting)
     - Vaccination records of existing pets
     - Yard photos (if relevant)

3. **Application Confirmation**
   - System sends email: "Application received"
   - Status: `PENDING`
   - Estimated review time: 2-5 business days
   - Application ID provided for tracking

#### **Platform Actions:**
- Validate all required fields
- Check for duplicate applications (same user + same pet)
- Notify shelter via email/dashboard alert
- Log submission timestamp
- Create application record in database

#### **Business Rules:**
- **One application per pet per user** (can't submit multiple for same pet)
- **Multiple applications allowed** (can apply to different pets)
- **Incomplete applications rejected** (all required fields must be filled)
- **Automatic rejection scenarios:**
  - User has been banned/blacklisted
  - User has abandoned previous adoptions
  - User is in "cooling-off period" (rejected within 30 days)

### **STAGE 2: Shelter Review**

#### **Shelter Actions:**

1. **Initial Review (24-48 hours)**
   - Change status: `PENDING` â†’ `UNDER_REVIEW`
   - Review submitted information
   - Platform displays:
     - Application summary
     - Red flags (if any):
       - Renting without landlord approval
       - No pet experience but requesting difficult breed
       - Work hours exceed recommended alone time
       - Existing pets not vaccinated
   - **Decision Matrix Aid** (platform provides):
     - Compatibility score (algorithm-based)
     - Risk factors highlighted
     - Similar past applications outcomes

2. **Reference Checks**
   - Platform provides templated forms to send references
   - System tracks reference responses
   - References can submit via secure link (no login needed)
   - **Reference Questions:**
     - How long have you known applicant?
     - Have you seen them with animals?
     - Responsible pet owner? (Yes/No with explanation)
     - Any concerns? (Free text)

3. **Veterinary Verification**
   - Platform auto-sends verification request to listed vet
   - Vet confirms:
     - Patient in good standing
     - Pets current on vaccinations
     - No animal welfare concerns
   - If no prior pets, flag for extra scrutiny (not rejection)

4. **Initial Decision**
   - **Option A: Approve for Visit**
     - Status: `UNDER_REVIEW` â†’ `APPROVED`
     - System sends email: "Congratulations! Next step: schedule visit"
     - Platform opens calendar booking system
   
   - **Option B: Request More Information**
     - Status remains `UNDER_REVIEW`
     - System sends email with specific questions
     - Applicant submits additional info
     - Review resumes
   
   - **Option C: Reject**
     - Status: `UNDER_REVIEW` â†’ `REJECTED`
     - **Rejection reason required** (dropdown + text):
       - Unsuitable living conditions
       - Insufficient pet experience
       - Failed reference check
       - Veterinary concerns
       - Does not meet pet's specific needs
       - Other (must explain)
     - System sends **compassionate rejection email** with:
       - Reason for rejection
       - Suggestions for improvement
       - Alternative pets that may be better match
       - Reapplication guidance (when allowed)
     - **Rejection cooling period:** 30 days before reapplying to same shelter

#### **Platform Actions During Review:**
- Track time spent in each status (analytics)
- Send reminder to shelter if no action after 48 hours
- Auto-notify applicant of status changes
- Log all shelter notes and decisions (audit trail)
- If pet becomes unavailable (adopted by another), auto-notify and offer similar pets

#### **Applicant Actions During Review:**
- **View application status** in dashboard
- **Withdraw application** if changed mind
  - Status: Any â†’ `WITHDRAWN`
  - No penalty, can reapply immediately to different pet
- **Respond to information requests**
- **Update contact information** if needed

### **STAGE 3: Visit Scheduling & Completion**

#### **Approved Application Next Steps:**

1. **Visit Scheduling**
   - **Who can schedule:**
     - Shelter proposes times (calendar availability set in settings)
     - Applicant books from available slots
     - OR shelter manually schedules after phone call
   
   - **Status Flow:**
     - `APPROVED` â†’ `VISIT_SCHEDULED`
   
   - **Platform Features:**
     - Integrated calendar with time zone handling
     - Automatic confirmation emails (both parties)
     - Reminder emails (24 hours before)
     - SMS reminders (if phone provided)
     - Reschedule option (up to 2 times, then must contact shelter)
     - Cancel option (applicant only, reverts to `APPROVED`)

2. **Visit Preparation**
   - **Email to Applicant Includes:**
     - Date, time, location
     - What to bring (ID, other pets if meeting required, family members)
     - What to expect (meet pet, Q&A, facility tour)
     - Dress appropriately (for playing with pet)
     - Questionnaire to prepare
   
   - **Email to Shelter Includes:**
     - Applicant details summary
     - Application highlights
     - Suggested discussion topics
     - Evaluation checklist

3. **The Visit (In-Person)**
   - **Duration:** 30-60 minutes typically
   - **Activities:**
     - Meet the pet (supervised interaction)
     - Walk/play session
     - If applicant has existing pets: controlled introduction (if safe)
     - Discuss pet's history, behavior, needs
     - Review adoption contract terms
     - Answer applicant questions
     - Assess applicant-pet compatibility
   
   - **Shelter Evaluation Checklist** (platform provides form):
     - How did pet react to applicant? (1-5 scale)
     - How did applicant handle pet? (1-5 scale)
     - Did applicant ask appropriate questions? (Yes/No)
     - Did applicant seem prepared? (Yes/No)
     - Any concerns? (Free text)
     - Recommendation: Proceed / Home check required / Reject

4. **Post-Visit Status Update**
   - **Shelter logs visit completion in platform:**
     - Status: `VISIT_SCHEDULED` â†’ `VISIT_COMPLETED`
     - Fill evaluation form
     - Upload visit notes
   
   - **Shelter Decision (within 48 hours):**
     
     **A. Proceed to Adoption (simple cases):**
     - Status: `VISIT_COMPLETED` â†’ `READY_FOR_ADOPTION`
     - If pet is low-maintenance, applicant is experienced, no red flags
     - Skip home check (shelter discretion)
     
     **B. Require Home Check (complex cases):**
     - Status: `VISIT_COMPLETED` â†’ `HOME_CHECK_PENDING`
     - **Trigger criteria (platform auto-suggests):**
       - First-time pet owner
       - High-energy dog or special needs pet
       - Children under 5 in household
       - Renting property
       - Yard safety concerns
       - Breed-specific requirements
     
     **C. Reject After Visit:**
     - Status: `VISIT_COMPLETED` â†’ `REJECTED_AFTER_VISIT`
     - Reason required (compassionate communication)
     - Suggest alternative pets if applicable

#### **Platform Actions:**
- Send status update notifications
- If no action after 48 hours, remind shelter
- Track visit outcome analytics
- Provide post-visit feedback form to applicant (optional)

## ðŸš« STATUS SKIP RULES

### **Cannot Skip (Mandatory Sequence):**

1. **Must Review Before Approving:**
   - `PENDING` â†’ cannot skip to `APPROVED` without `UNDER_REVIEW`
   - Reason: Compliance, documentation, reference checks

2. **Must Complete Visit Before Final Approval:**
   - `APPROVED` â†’ cannot skip to `READY_FOR_ADOPTION` without visit statuses
   - Exception: Emergency situations (admin override only)

3. **If Home Check Required, Cannot Skip:**
   - Once `HOME_CHECK_PENDING` set, must complete full home check sequence
   - Cannot jump to `READY_FOR_ADOPTION` directly

4. **Must Finalize Before Adopted:**
   - `READY_FOR_ADOPTION` â†’ cannot skip to `ADOPTION_COMPLETED` without `ADOPTED`
   - Reason: Contract signature, payment, handover documentation

### **Can Skip (Optional Steps):**

1. **Home Check (Shelter Discretion):**
   - `VISIT_COMPLETED` â†’ can go directly to `READY_FOR_ADOPTION`
   - Skip if: experienced adopter, suitable living situation verified, low-risk pet

2. **Under Review (Rare):**
   - Admin override: `PENDING` â†’ `APPROVED` in urgent cases
   - Example: Returning adopter with perfect history
   - Requires audit note

3. **Visit Rescheduling:**
   - Can go back from `VISIT_SCHEDULED` to `APPROVED` if cancelled
   - No penalty, normal flow resumes

### **Backwards Movement (Limited):**

- **Cannot go backwards** from terminal states (`REJECTED`, `WITHDRAWN`, `ADOPTION_COMPLETED`)
- **Can request new application** but old one remains closed
- **Exception:** Admin can reopen `REJECTED` applications with justification note
